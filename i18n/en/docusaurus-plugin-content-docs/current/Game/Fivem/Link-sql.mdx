# Configuring the MySQL Connection String for a FiveM Server

This page provides instructions on updating the MySQL connection string to be compatible with `oxmysql` and `mysql-async`, specifically for configuring a FiveM server.

## Obsolete Connection String

The following connection string is now obsolete for `oxmysql`:

```text
set mysql_connection_string "server=IP_PHP_SANS_PORTS;database=NOM;userid=NOM_UTILISATEUR_PHP;password=MOT_DE_PASSE_PHP"
```

This format may not work with the latest versions of `oxmysql` and requires an update.

## Updated Connection String

For compatibility with `oxmysql` and `mysql-async`, use the following connection string:

```text
set mysql_connection_string "mysql://NOM_UTILISATEUR_PHP:MOT_DE_PASSE_PHP@IP_PHP_SANS_PORTS/NOM?charset=utf8mb4"
```

Replace `NOM_UTILISATEUR_PHP`, `MOT_DE_PASSE_PHP`, `IP_PHP_SANS_PORTS`, and `NOM` with your own information. Ensure that the password does not contain the '@' symbol.

## Use for FiveM Server

- Add this string to the `server.cfg` file of your FiveM server for database connection.

## Connection Testing

- After updating the configuration, restart your FiveM server to apply the changes.
- Ensure that your FiveM server can correctly connect to the MySQL database.
- If there are any issues, recheck the connection information and the settings of your MySQL server.
